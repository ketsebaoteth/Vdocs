# Themeing Vdocs Documentation

Themeing in a vdocs documentation is done through the `global.css` file located in the `vdocs` project's **./assets/css/** directory. This file is used to manage the site's appearance and is recommended to use CSS variables for colors and styles.
if you go to the file you will find a collection of css variables for different themes, you can modify these variables to create your own theme.
the css variables you find will look somewhat like this

 you might find this familier if you are already expereanced with shadcn, if not what is happening is that we are defining a set of css variables for the light theme and then we are defining a set of css variables for the dark theme, and then we are using these variables in the css file to style the site.
 the color variables are in the hsv color space, this is because hsv is a more intuitive color space to work with than rgb, you can use a color picker to get the hsv values of a color and then use them in the css file.
